## কেন OpenTaint

AI আজকের নিরাপত্তা টুলিং যতটা সামলাতে পারে তার চেয়ে দ্রুত প্রোডাকশন কোড তৈরি করে।

LLM নিরাপত্তা এজেন্ট মানুষ যা মিস করে সেই দুর্বলতা খুঁজে পায়, প্রতিটি ফাইলে টোকেন খরচ করে, এবং তারপরও সবকিছু ধরতে পারার গ্যারান্টি দিতে পারে না।

AI যত বেশি কোড লেখে, আপনার তত বেশি ভিত্তি হিসেবে formal methods প্রয়োজন।

- **প্যাটার্ন ম্যাচিং ইঞ্জিন যা মিস করে তা খুঁজে বের করুন।** inter-procedural dataflow ইঞ্জিন ফাংশন সীমানা, পার্সিস্টেন্স লেয়ার, অ্যালিয়াস এবং অ্যাসিঙ্ক কোড জুড়ে অবিশ্বস্ত ডেটা ট্র্যাক করে।
- **একটি আবিষ্কার সম্পূর্ণ কভারেজে পরিণত হয়।** কোড-নেটিভ নিয়ম আপনাকে প্রতিটি আবিষ্কৃত দুর্বলতাকে একটি নিয়ম হিসেবে কার্যকর করতে দেয় যেখানে ইঞ্জিন এটি সম্পূর্ণ কোডবেস জুড়ে নির্ধারণমূলকভাবে, মিনিটের CPU সময়ে প্রয়োগ করে।
- **ওপেন সোর্স, সবকিছু অন্তর্ভুক্ত।** ইঞ্জিন, নিয়ম, CI ইন্টিগ্রেশন — সম্পূর্ণ স্ট্যাক Apache 2.0 এবং MIT লাইসেন্সের অধীনে প্রকাশিত। টেইন্ট ট্র্যাকিং আনলক করতে কোনো পেইড টিয়ার নেই, নিজের নিয়ম লেখার উপর কোনো বাধা নেই।

## দ্রুত শুরু

**ইনস্টল স্ক্রিপ্ট (Linux/macOS)**
```
curl -fsSL https://raw.githubusercontent.com/seqra/opentaint/main/scripts/install/install.sh | bash
```

**Homebrew দিয়ে ইনস্টল (Linux/macOS):**
```bash
brew install --cask seqra/tap/opentaint
```

**ইনস্টল স্ক্রিপ্ট (Windows PowerShell)**
```
irm https://raw.githubusercontent.com/seqra/opentaint/main/scripts/install/install.ps1 | iex
```

**আপনার প্রোজেক্ট স্ক্যান করুন:**
```bash
opentaint scan
```

**অথবা Docker ব্যবহার করুন:**
```bash
docker run --rm -v $(pwd):/project -v $(pwd):/output \
  ghcr.io/seqra/opentaint:latest \
  opentaint scan --output /output/results.sarif /project
```

আরও বিকল্পের জন্য দেখুন [ইনস্টলেশন](../../docs/README.md#installation) এবং [ব্যবহার](../../docs/README.md#usage)।
